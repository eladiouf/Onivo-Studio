# Live Operations Cadence

## Session Open

Every serious work session should begin by reading:

- the connected project bridge
- the project dashboard
- the latest strategy review for the target project
- `Mission Radar`
- `Risk Register`

## During Execution

- work should attach to an existing mission or create a new one
- structural changes should create or update a decision note
- new blockers should become explicit risks

## Session Close

Before closing a meaningful work session:

- update mission status
- update next review dates when needed
- record verification evidence or blockers
- update release readiness if publication surfaces were affected

## Weekly Strategy Rhythm

- refresh the current project strategy review
- retire stale risks
- open new missions only when they clarify the next execution slice
