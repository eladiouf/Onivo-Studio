# Start Here - Obsidian

This repository is the Obsidian vault.

## Open The Vault

1. Install `Obsidian` on your machine.
2. In Obsidian, choose `Open folder as vault`.
3. Select the repository root:
   `C:\Users\El Hadji\Downloads\STAT_MOD\test\MOD_MINECRAFT_STAT\.tmp-onivo-audit`
4. Open [[Studio Home]].

## What To Use First

- `Studio Home` for global navigation
- `Current Portfolio` for active projects
- `Operations Index` for how the studio runs
- `Decision Index` for major choices
- `Release Board` for publishing readiness

## Simple Daily Workflow

1. Open [[Studio Home]].
2. Jump to the project or operating area you need.
3. Add or update notes in the relevant vault section.
4. Commit the repository after meaningful studio changes.

## Folder Logic

- `vault/00 Home` central entry points
- `vault/01 Governance` studio rules and authority
- `vault/02 Strategy` long-term direction
- `vault/03 Operations` execution system and playbooks
- `vault/04 Projects` project-specific navigation
- `vault/05 Research` findings, exploration, references
- `vault/06 Visual` art direction and asset thinking
- `vault/07 Publishing` release readiness and channels
- `vault/08 Decisions` durable decisions
- `vault/09 Postmortems` retrospectives and failures

## Practical Rules

- Keep one note per durable idea or decision.
- Prefer links between notes instead of repeating text.
- Put project-specific material under `vault/04 Projects` unless it is reusable studio knowledge.
- Use the repo root as the single source of truth for studio memory.
